# Количественный анализ результативности интервенций
Репозиторий содержит воспроизводимый Jupyter Notebook с количественным приложением к дипломной работе о результативности военных интервенций.

Основной файл:

- `Diplom_analysis.ipynb`

Notebook последовательно выполняет:

1. загрузку и подготовку данных;
2. проверку баланса выборки;
3. анализ связи декларации целей и результативности;
4. анализ латентного успеха и незаявленных целей;
5. сравнение референтных субъектов;
6. анализ контекстуальных факторов;
7. блок международной поддержки и внимания;
8. геополитическую декомпозицию;
9. сравнение периодов до/после 2008 года;
10. OLS и ordinal logit модели;
11. сохранение итоговых таблиц и графиков.

## Ожидаемая структура проекта

Данные можно положить либо в корневую папку проекта, либо в папку `data/`.

Минимально ожидаемые файлы:

- `Results_table-3.xlsx` — основная таблица с целями и результативностью;
- файл стороны вмешательства с названием, содержащим `side`, `iside`, `interns` или близкие варианты;
- `merged_dataset_improved.csv` — показатели поддержки и объективного гуманитарного кризиса;
- `Копия hum_crisis.xlsx` — субъективный гуманитарный кризис;
- `Копия ethics_copy.xlsx` — поддержка, внимание и международно-правовые переменные.

Notebook создает результаты в:

- `outputs/tables/` — итоговые таблицы `.csv`;
- `outputs/figures/` — итоговые графики `.png`.

## Запуск

Создайте окружение и установите зависимости:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Запустите notebook сверху вниз в Jupyter или выполните из командной строки:

```bash
jupyter nbconvert --to notebook --execute DIPLOMA_ANALYSIS_GITHUB_CLEAN.ipynb --inplace
```

## Воспроизводимость

В notebook используются относительные пути. Год разделения исторических периодов задается параметром `HISTORICAL_SPLIT_YEAR = 2008` в первом блоке настроек.

Шкалы результативности 1–5 трактуются как порядковые: для основных сравнений используются Spearman, Kruskal-Wallis и Mann-Whitney; OLS и ordinal logit применяются как модельные проверки.
